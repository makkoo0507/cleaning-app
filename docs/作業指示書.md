## 3. 環境変数

### next/.env.local（ローカル）

```env
NEXT_PUBLIC_SUPABASE_URL=http://127.0.0.1:54321
NEXT_PUBLIC_SUPABASE_ANON_KEY=
SUPABASE_SERVICE_ROLE_KEY=
NEXT_PUBLIC_LIFF_ID=
VENDOR_PASSWORD_BASE=localdev   # ベンダーアカウント初期パスワードのベース値
```

### Vercel 環境変数（本番）

| 変数名 | 用途 |
|---|---|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase Project URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabase anon key |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase service role key（サーバーのみ） |
| `NEXT_PUBLIC_LIFF_ID` | LINE LIFF アプリ ID |
| `VENDOR_PASSWORD_BASE` | ベンダーアカウント初期パスワードのベース値（本番用） |

> ベンダーアカウントの初期パスワード = `{VENDOR_PASSWORD_BASE}+{slug}`

## 6. LINE 通知実装メモ

Channel Access Token は業者ごとに DB から取得する。

```typescript
// lib/line.ts
export async function pushMessage(
  channelAccessToken: string,
  lineUserId: string,
  text: string
) {
  await fetch('https://api.line.me/v2/bot/message/push', {
    method: 'POST',
    headers: {
      Authorization: `Bearer ${channelAccessToken}`,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({ to: lineUserId, messages: [{ type: 'text', text }] }),
  });
}
```

## 7. RLS ポリシー方針

- JWT の `app_metadata.contractor_id` / `app_metadata.role` を `my_company_id()` / `my_role()` で参照
- 管理者相当（`contractor_admin` / `contractor_vendor`）は全 CRUD 可
- `contractor_viewer` は参照のみ
- `platform_admin` は service_role クライアントで操作（RLS 対象外）
