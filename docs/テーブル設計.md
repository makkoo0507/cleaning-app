# テーブル設計

**プロジェクト名:** 民泊清掃管理アプリ  
**作成日:** 2026-06-20  
**バージョン:** 1.0（2026-06-24 統合・整理）

---

## テナント分離の方針

全テーブル（cleaning_records を除く）に `contractor_id` を持たせ、RLS で**自社データのみアクセス可能**に制限する。

---

## テーブル一覧

| テーブル名 | 概要 |
| --- | --- |
| contractors | 業者テナント |
| users | 全ユーザー共通（認証・通知・ロール） |
| contractor_member_profiles | 業者スタッフ固有の情報（users の拡張） |
| cleaner_profiles | 清掃者固有の情報（users の拡張） |
| property_member_profiles | 物件関係者固有の情報（users の拡張） |
| properties | 民泊物件 |
| property_members | 物件ごとの関係者・役割・通知設定 |
| jobs | 清掃案件（スケジュール） |
| cleaning_records | 清掃記録 |
| features | オプション機能カタログ |
| contractor_features | 業者ごとのオプション加入状況 |
| contractor_notification_settings | 業者ごとの通知設定 |
| reminder_logs | リマインド送信ログ（重複送信防止） |

## users（ユーザー共通）

認証・通知・ロール判定に必要な共通情報のみ保持する。役割固有の情報は拡張テーブルで管理する。

| カラム名 | 型 | 制約 | コメント |
| --- | --- | --- | --- |
| id | uuid | PK, NOT NULL | Supabase Auth の user id と一致 |
| contractor_id | uuid | FK → contractors.id, NULL 許容 | 所属テナント（platform_admin は NULL） |
| role | text | NOT NULL | 下記参照 |
| name | text | NOT NULL | 表示名 |
| line_user_id | text | NULL 許容 | LINE ユーザー ID（通知送信用） |
| invite_token | uuid | NULL 許容, UNIQUE | 招待 URL 用トークン。紐付け完了後に NULL クリア |
| created_at | timestamptz | NOT NULL, DEFAULT now() | — |

### users.role の値

| 値 | 説明 |
| --- | --- |
| `platform_admin` | ベンダー運営（`/vendor` のみ利用。contractor_id = null） |
| `contractor_admin` | 業者管理者（全操作可） |
| `contractor_viewer` | 業者閲覧者（閲覧のみ） |
| `contractor_vendor` | ベンダーアカウント（運営が業者管理画面に入るための隠しアカウント） |
| `cleaner` | 清掃者（LINE Mini App） |
| `contact` | 物件関係者・オーナー（LINE Mini App） |

## プラン上限チェック

新規登録時にアプリ側で以下を確認し、上限超過の場合はアップグレード案内を表示する。

```typescript
// 物件追加前チェック
const count = await db.properties.count({ where: { contractor_id } });
if (contractor.max_properties !== null && count >= contractor.max_properties) {
  throw new Error('PLAN_LIMIT_PROPERTIES');
}

// 清掃者追加前チェック
const count = await db.users.count({ where: { contractor_id, role: 'cleaner' } });
if (contractor.max_cleaners !== null && count >= contractor.max_cleaners) {
  throw new Error('PLAN_LIMIT_CLEANERS');
}
```

## RLS 方針

- JWT の `app_metadata.contractor_id` と `app_metadata.role` を `my_company_id()` / `my_role()` ヘルパー関数で取り出してポリシーに使用
- 管理者相当ロール（`contractor_admin` / `contractor_vendor`）は全 CRUD 可
- `platform_admin` は RLS 対象外（service_role クライアントで操作）
- `contractor_viewer` は参照のみ
