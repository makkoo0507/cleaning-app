# テーブル設計

**プロジェクト名:** 民泊清掃管理アプリ  
**作成日:** 2026-06-20  
**バージョン:** 1.7

---

## テナント分離の方針

全テーブル（cleaning_records を除く）に `company_id` を持たせ、RLS で**自社データのみアクセス可能**に制限する。

---

## テーブル一覧

| テーブル名 | 概要 |
| --- | --- |
| contractor_companies | 清掃業者テナント |
| users | 全ユーザー共通（認証・通知・ロール） |
| contractor_member_profiles | 業者テナント関係者固有の情報（users の拡張） |
| cleaner_profiles | 清掃者固有の情報（users の拡張） |
| property_member_profiles | 物件関係者固有の情報（users の拡張） |
| properties | 民泊物件 |
| property_members | 物件ごとの関係者・役割・通知設定 |
| jobs | 清掃案件（スケジュール） |
| cleaning_records | 清掃記録 |

---

## contractor_companies（清掃業者テナント）

| カラム名 | 型 | 制約 | コメント |
| --- | --- | --- | --- |
| id | uuid | PK, NOT NULL | — |
| name | text | NOT NULL | 会社名 |
| plan | text | NOT NULL, DEFAULT 'free' | 'free' または 'paid' |
| max_properties | int | NULL 許容 | フリー: 1 / 有料: NULL（無制限） |
| max_cleaners | int | NULL 許容 | フリー: 5 / 有料: NULL（無制限） |
| created_at | timestamptz | NOT NULL, DEFAULT now() | — |

---

## users（ユーザー共通）

認証・通知・ロール判定に必要な共通情報のみ保持する。役割固有の情報は拡張テーブルで管理する。

| カラム名 | 型 | 制約 | コメント |
| --- | --- | --- | --- |
| id | uuid | PK, NOT NULL | Supabase Auth の user id と一致 |
| company_id | uuid | FK → contractor_companies.id, NOT NULL | 所属テナント |
| role | text | NOT NULL | 'contractor_admin' / 'contractor_staff' / 'cleaner' / 'contact' |
| name | text | NOT NULL | 表示名 |
| line_user_id | text | NULL 許容 | LINE ユーザー ID（通知送信用） |
| invite_token | uuid | NULL 許容, UNIQUE | 招待 URL 用トークン。紐付け完了後に NULL クリア |
| created_at | timestamptz | NOT NULL, DEFAULT now() | — |

---

## contractor_member_profiles（業者テナント関係者プロフィール）

`users.role = 'contractor_admin' または 'contractor_staff'` のユーザーに対応する拡張テーブル。

| カラム名 | 型 | 制約 | コメント |
| --- | --- | --- | --- |
| user_id | uuid | PK, FK → users.id, NOT NULL | 対象ユーザー |
| department | text | NULL 許容 | 部署名 |
| employee_code | text | NULL 許容 | 社員番号 |
| note | text | NULL 許容 | 備考 |
| created_at | timestamptz | NOT NULL, DEFAULT now() | — |

---

## cleaner_profiles（清掃者プロフィール）

`users.role = 'cleaner'` のユーザーに対応する拡張テーブル。今後、清掃者固有の情報が増えた場合にカラムを追加する。

| カラム名 | 型 | 制約 | コメント |
| --- | --- | --- | --- |
| user_id | uuid | PK, FK → users.id, NOT NULL | 対象ユーザー |
| skills | text | NULL 許容 | 得意な清掃種別・資格など |
| note | text | NULL 許容 | 備考 |
| created_at | timestamptz | NOT NULL, DEFAULT now() | — |

---

## property_member_profiles（物件関係者プロフィール）

`users.role = 'contact'` のユーザーに対応する拡張テーブル。今後、関係者固有の情報が増えた場合にカラムを追加する。

| カラム名 | 型 | 制約 | コメント |
| --- | --- | --- | --- |
| user_id | uuid | PK, FK → users.id, NOT NULL | 対象ユーザー |
| company_name | text | NULL 許容 | 所属会社名 |
| phone | text | NULL 許容 | 連絡先電話番号 |
| billing_address | text | NULL 許容 | 請求先住所 |
| created_at | timestamptz | NOT NULL, DEFAULT now() | — |

| カラム名 | 型 | 制約 | コメント |
| --- | --- | --- | --- |
| id | uuid | PK, NOT NULL | — |

| address | text | NOT NULL | 住所 |
| notes | text | NULL 許容 | 特記事項 |

## property_members（物件関係者紐付け）

`property_member_profiles` のユーザーと物件を紐づけるテーブル。1人が複数物件を担当できる。

| カラム名 | 型 | 制約 | コメント |
| --- | --- | --- | --- |
| user_id | uuid | PK(複合), FK → users.id, NOT NULL | 関係者 |
| property_id | uuid | PK(複合), FK → properties.id, NOT NULL | 対象物件 |
| role | text | NOT NULL | 'owner' / 'operations' / 'sales' など |
| notify | boolean | NOT NULL, DEFAULT true | LINE 通知を受け取るか |
| created_at | timestamptz | NOT NULL, DEFAULT now() | — |

### property_members.role の値（例）

| 値 | 意味 |
| --- | --- |
| owner | オーナー（物件の所有者） |
| operations | 運用担当（日常管理を担う） |
| sales | 営業（契約・問い合わせ窓口） |

## プラン上限チェック

新規登録時にアプリ側で以下を確認し、上限超過の場合はアップグレード案内を表示する。

```typescript
// 物件追加前チェック
const count = await db.properties.count({ where: { company_id } });
if (company.max_properties !== null && count >= company.max_properties) {
  throw new Error('PLAN_LIMIT_PROPERTIES');
}

// 清掃者追加前チェック
const count = await db.users.count({ where: { company_id, role: 'cleaner' } });
if (company.max_cleaners !== null && count >= company.max_cleaners) {
  throw new Error('PLAN_LIMIT_CLEANERS');
}
```
