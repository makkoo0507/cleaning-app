# UI ガイドライン

**プロジェクト名:** 民泊清掃管理アプリ  
**作成日:** 2026-06-27

---

## 1. 保存ボタンの状態表示

保存ボタンは、送信中に「保存中…」テキストへ変化し、`disabled` にする。

**実装例（ユーザー管理 StaffForm より）:**

```tsx
<button
  type="submit"
  disabled={pending}
  className="... disabled:opacity-50"
>
  {pending ? "保存中…" : "保存"}
</button>
```

**ルール:**
- テキスト: `保存` → `保存中…`（三点リーダーは `…` U+2026 を使用）
- ボタンを `disabled` にして二重送信を防ぐ
- `disabled:opacity-50` で視覚的に非活性を示す

---

## 2. 保存後の挙動

保存成功後はページ遷移せず、同じページに留まる。フォームの下に完了メッセージを表示する。

**実装例（LINE設定 SettingsForm より）:**

```tsx
{state.error && (
  <p className="text-sm text-red-600" role="alert">
    {state.error}
  </p>
)}
{state.success && (
  <p className="text-sm text-green-600">保存しました。</p>
)}
```

**ルール:**

- 保存成功後はリダイレクトしない
- 成功メッセージ: `保存しました。` をフォーム内（送信ボタンの上）にインライン表示
- 文字色: `text-green-600`
- エラーメッセージ: `text-red-600`、`role="alert"` を付与
- エラーと成功メッセージは同じ位置に出す（同時には表示しない）
- **新規作成の場合は例外**: 保存後、作成されたデータの編集ページ（`/[resource]/[id]/edit`）へリダイレクトする

---

## 3. 削除確認ダイアログ

削除ボタンを押したら、即削除せず確認ダイアログを表示する。

**ルール:**

- 削除ボタン押下 → 確認ダイアログ表示 → ユーザーが「削除」を選択 → 削除実行
- ダイアログには削除対象の名前を含める（例: 「"山田太郎" を削除しますか？」）
- ダイアログのボタン: 「削除」（赤系）と「キャンセル」の2択
- 「削除」は取り消せない操作であることをテキストで明示する

---

## 4. バリデーションエラーの表示タイミング

エラーはサーバーサイドのバリデーション結果のみ表示する。

**ルール:**

- クライアント側のリアルタイムバリデーション（入力中・フォーカスアウト時）は行わない
- エラーは保存ボタン押下後、サーバーのレスポンスを受けてフォーム下部にインライン表示する
- HTML の `required` など最低限のブラウザ標準バリデーションは使用してよい
