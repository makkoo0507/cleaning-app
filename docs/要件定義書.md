# 要件定義書

**プロジェクト名:** 民泊清掃管理アプリ  
**作成日:** 2026-06-20  
**バージョン:** 1.0（2026-06-24 統合・整理）

---

## 1. プロジェクト概要

### 1.1 背景・目的

民泊清掃業務において、業者・民泊オーナー・フリーランス清掃者の3者間のコミュニケーションや業務管理が、現状では電話・メール・LINE の個別やりとりに頼っており非効率な状態にある。

本システムにより、スケジュール管理・通知・清掃記録の共有を一元化し、業務効率と品質を向上させる。

将来的には複数の業者に提供する SaaS として展開する。1社目は開発・テスト運用とし、2社目以降からフリーミアム型の商用サービスとして公開する。

### 1.2 プランと料金体系

プランは**物件数・清掃者数の登録上限**を決める。

| プラン | 対象 | 民泊物件数上限 | 清掃者数上限 | 月額 |
| --- | --- | --- | --- | --- |
| **フリープラン** | 2社目以降・導入初期 | 1件 | 5名 | 無料 |
| **有料プラン** | 上限を超えた場合 | 無制限 | 無制限 | 別途設定 |

- 上限に達した場合、新規登録をブロックしてアップグレードを促す
- プラン管理はベンダーがデータベース上で行う（DB の `contractors.plan` カラムで制御）

### 1.2.1 有料オプション

プランとは独立した**追加機能**。ベンダーが業者ごとに `contractor_features` テーブルで ON/OFF を管理する。

| オプション | 機能概要 |
| --- | --- |
| **請求・支払い** | 案件ごとの請求額・支払い額の記録、月次集計、CSV エクスポート |

- 加入・解約はベンダー（システム管理者）が対応する
- 業者管理者は「設定 > オプション」で加入状況を確認できる
- 未加入の業者が該当画面を開くと「システム管理者へお申し込みください」と案内される

### 1.3 登場人物（アクター）

| アクター | ロール | 役割 |
| --- | --- | --- |
| **ベンダー（システム運営）** | `platform_admin` | `/vendor` から業者アカウント発行・オプション管理を行う。全業者の管理画面にも入れる |
| **業者（管理者）** | `contractor_admin` | 自社テナント内の全機能へのフルアクセス。物件・清掃者・オーナー・スケジュール管理、請求・支払い記録、ユーザー管理 |
| **業者（閲覧者）** | `contractor_viewer` | 物件・スケジュール・清掃記録・請求の閲覧のみ |
| **ベンダーアカウント** | `contractor_vendor` | ベンダーが業者管理画面に入るための隠しアカウント。管理者相当の権限 |
| **民泊オーナー** | `contact` | 自社テナントの清掃スケジュールの確認、清掃記録の閲覧（LINE Mini App） |
| **清掃者（フリーランス）** | `cleaner` | 担当スケジュールの確認、清掃記録（完了時刻・所要時間）の登録（LINE Mini App） |

## 2. システム構成

### 2.1 プラットフォーム

| 画面 | 対象 | 技術 |
|---|---|---|
| **ベンダー管理画面** | ベンダー運営 | ブラウザ（`/vendor`） |
| **管理 Web アプリ** | 業者 | ブラウザ（PC 主体） |
| **LINE Mini App (LIFF)** | 清掃者・民泊オーナー | LINE アプリ内 Web ビュー |

### 2.2 通知

- **LINE Messaging API** を使用
- 業者がそれぞれ自社の LINE 公式アカウントを開設し、Channel Access Token / Channel Secret を本システムに登録する
- 清掃者・民泊オーナーは各業者の LINE 公式アカウントを友だち追加して通知を受信する
- 通知の送信元は業者ごとに異なる（業者ブランドで送信される）

### 2.3 LINE アカウントの二層構造

| レイヤー | 誰のアカウント | 役割 |
|---|---|---|
| **LINE Login / LIFF** | **ベンダー共通（1 つ）** | ログイン・本人確認、LINE Mini App 画面の入口 |
| **Messaging API（公式アカウント）** | **各業者ごと** | 通知メッセージの送信元。業者ブランドで届く |

## 3. 機能要件

### 3.1 業者（管理 Web アプリ）

画面のナビゲーション構成：

```
ダッシュボード
スケジュール
清掃記録
請求・支払い（オプション加入時のみ）
基本情報 ▶︎
  物件
  オーナー
  清掃者
設定 ▶︎（管理者のみ）
  LINE連携
  通知設定
  オプション
  ユーザー管理
```

#### 3.1.1 物件管理

- 民泊物件の登録・編集・削除（物件名、住所、特記事項）
- 物件ごとに複数の関係者を登録・管理
- 関係者ごとに LINE 通知の ON/OFF を設定できる

#### 3.1.2 オーナー管理

- 関係者（オーナー・運用担当・営業など）の登録・編集・削除
- 各ユーザーへの LINE アカウント紐付け

#### 3.1.3 清掃者管理

- 清掃者（フリーランス）の登録・編集・削除
- 各ユーザーへの LINE アカウント紐付け

#### 3.1.4 スケジュール管理

- 清掃案件の作成（対象物件、清掃日時、担当清掃者のアサイン）
- スケジュールの一覧表示・編集・削除

#### 3.1.5 通知送信

- スケジュール作成時に、担当清掃者と物件の通知有効な関係者へ LINE 通知を自動送信
- 清掃前日・当日に清掃者・オーナーへリマインド通知を自動送信（業者ごとに送信先・タイミングを設定可能）
- 清掃完了時に物件の通知有効な関係者へ完了通知を自動送信

#### 3.1.6 清掃記録閲覧

- 全案件の清掃記録（完了時刻・所要時間）を一覧表示

#### 3.1.7 請求・支払い管理（オプション）

- 清掃案件ごとの金額（請求額・支払い額）を記録
- 月次・期間別の集計表示
- CSV エクスポート機能

#### 3.1.8 ユーザー管理（管理者のみ）

- スタッフアカウントの登録・編集・削除
- 権限：管理者 / 閲覧者

#### 3.1.9 設定

- LINE Messaging API のトークン・チャネルシークレット登録
- リマインド通知の送信先・タイミング設定
- オプション機能の利用 ON/OFF

---

### 3.2 清掃者（LINE Mini App）

- 自分がアサインされた清掃案件の一覧・詳細表示
- 清掃開始・完了時刻の記録、メモ入力

---

### 3.3 民泊オーナー（LINE Mini App）

- 自物件に紐づいた清掃案件の一覧表示
- 清掃完了後の記録（完了時刻・所要時間・メモ）を閲覧

---

## 4. 通知仕様

| タイミング | 送信先 | 内容 |
|---|---|---|
| スケジュール作成時 | 清掃者 | 「〇月〇日 〇〇物件の清掃が入りました」|
| スケジュール作成時 | 物件の通知有効な関係者 | 「〇月〇日に清掃が予定されました」|
| 清掃前日（20:00頃） | 清掃者・オーナー（設定による） | 翌日の清掃リマインド |
| 清掃当日（8:00頃） | 清掃者・オーナー（設定による） | 当日の清掃リマインド |
| 清掃完了時 | 物件の通知有効な関係者 | 「清掃が完了しました（完了時刻・所要時間）」|

---

## 5. 非機能要件

### 5.1 スケール

| フェーズ | テナント数 | サーバー構成 |
| --- | --- | --- |
| **開発・テスト** | 1社（自社） | Vercel Hobby + Supabase Free（無料） |
| **商用開始** | 2社〜 | Vercel Pro + Supabase Pro |

### 5.2 セキュリティ

- 管理 Web アプリはメール＋パスワード認証（業者のみ）
- ログインは業者別URL `/{slug}/login` のみ（汎用ログインは廃止）
- 各ユーザーは自分に紐づくデータのみ閲覧可能（Supabase RLS によるテナント分離）
- LINE Messaging API のトークン・シークレットはカラムレベルセキュリティで保護
