# 要件定義書

**プロジェクト名:** 民泊清掃管理アプリ  
**作成日:** 2026-06-20  
**バージョン:** 1.3（2026-06-21 更新: ログインを業者別URL `/{slug}/login` に一本化）

---

## 1. プロジェクト概要

### 1.1 背景・目的

民泊清掃業務において、業者・民泊オーナー・フリーランス清掃者の3者間のコミュニケーションや業務管理が、現状では電話・メール・LINE の個別やりとりに頼っており非効率な状態にある。

本システムにより、スケジュール管理・通知・清掃記録の共有を一元化し、業務効率と品質を向上させる。

将来的には複数の業者に提供する SaaS として展開する。1社目は開発・テスト運用とし、2社目以降からフリーミアム型の商用サービスとして公開する。

### 1.2 プランと料金体系

| プラン | 対象 | 民泊物件数上限 | 清掃者数上限 | 月額 |
| --- | --- | --- | --- | --- |
| **フリープラン** | 2社目以降・導入初期 | 1件 | 5名 | 無料 |
| **有料プラン** | 上限を超えた場合 | 無制限 | 無制限 | 別途設定 |

- 1社目は開発・テスト用途として扱い、プラン管理の対象外とする
- 上限に達した場合、新規登録をブロックしてアップグレードを促す

---

### 1.3 登場人物（アクター）

| アクター | 役割 |
| --- | --- |
| **プラットフォーム運営者** | 全テナント（業者）の管理、プラン変更。開発・運営者自身が担当（専用画面または直接 DB 操作） |
| **業者（管理者）** | 自社テナント内の全機能へのフルアクセス。民泊オーナーとの契約管理、清掃者のアサイン、スケジュール作成・管理、請求・支払い記録の管理、社員の管理 |
| **業者（社員）** | 物件・清掃者・オーナー・スケジュールの管理、清掃記録の閲覧、請求・支払いの閲覧。管理者専用の「社員管理（権限変更）」と「設定 > LINE連携」以外はほぼ全機能を利用可能（現行実装。当初想定の「閲覧のみ」より広い権限） |
| **民泊オーナー** | 自社テナントの清掃スケジュールの確認、清掃記録の閲覧 |
| **清掃者（フリーランス）** | 担当スケジュールの確認、清掃記録（完了時刻・所要時間）の登録 |

## 2. システム構成

### 2.1 プラットフォーム

| 画面 | 対象 | 技術 |
|---|---|---|
| **管理 Web アプリ** | 業者 | ブラウザ（PC 主体） |
| **LINE Mini App (LIFF)** | 清掃者・民泊オーナー | LINE アプリ内 Web ビュー |

### 2.2 通知

- **LINE Messaging API** を使用
- 業者がそれぞれ自社の LINE 公式アカウントを開設し、Channel Access Token / Channel Secret を本システムに登録する
- 清掃者・民泊オーナーは各業者の LINE 公式アカウントを友だち追加して通知を受信する
- 通知の送信元は業者ごとに異なる（業者ブランドで送信される）

### 2.3 LINE アカウントの二層構造（重要）

清掃者・オーナー側では **2 種類の LINE アカウント** が別レイヤーで関わる。混同しないこと。

| レイヤー | 誰のアカウント | 役割 |
|---|---|---|
| **LINE Login / LIFF** | **ベンダー共通（1 つ）** | ログイン・本人確認、LINE Mini App 画面の入口。全業者で共通の LIFF アプリ（清掃者用・オーナー用の 2 つ）を使う |
| **Messaging API（公式アカウント）** | **各業者ごと** | 通知メッセージの送信元。業者ブランドで届く |

- **認証は全業者共通の LIFF**、**通知は業者ごとの公式アカウント**、という分担になる。
- そのため「通知を受け取る」には、ユーザーは **自分の担当業者の公式アカウント** を友だち追加している必要がある。
- LIFF の友だち追加機能はリンクした 1 アカウントしか追加できず、業者ごとの公式アカウントは追加させられないため、**業者公式アカウントの友だち追加は LIFF とは別導線**（招待時に各業者が自社の友だち追加 URL/QR を案内、リッチメニュー等）で行う。

#### 清掃者・オーナーが LIFF（Mini App）を開く導線

| タイミング | 開き方 | リンクの送り主 |
|---|---|---|
| 初回（LINE 紐付け） | 招待 URL `/liff/invite?token=...` をタップ → ロールに応じた LIFF でログインし `line_user_id` を紐付け | 業者が LINE/SMS で送付（BF-02）|
| 以降（日常） | 通知メッセージ内のリンク `https://liff.line.me/{LIFF ID}` をタップ → LINE 内で Mini App が開く | **各業者の公式アカウント**（スケジュール通知・前日リマインド等）|

> つまり「**業者の公式アカウントがリンクを配り、そのリンクでベンダー共通の LIFF が開く**」二層構造。LIFF アプリ自体は特定の公式アカウントに属さず、`liff.line.me/{ID}` のリンクからどのトークでも開ける。再アクセスを楽にするなら各業者の公式アカウントにリッチメニューを設定してもよい（任意）。

## 3. 機能要件

### 3.1 業者（管理 Web アプリ）

#### 3.1.0 社員管理（管理者のみ）

- 社員アカウントの登録・編集・削除
- 社員ごとの権限は固定（スケジュール管理・清掃記録閲覧は可、請求管理・ユーザー管理は閲覧のみ）

#### 3.1.1 物件管理

- 民泊物件の登録・編集・削除
  - 物件名、住所、特記事項
- 物件ごとに複数の関係者を登録・管理（物件関係者管理）
  - 関係者の追加・削除、役割（オーナー / 運用担当 / 営業 など）の設定
  - 関係者ごとに LINE 通知の ON/OFF を設定できる

#### 3.1.2 ユーザー管理

- 関係者（オーナー・運用担当・営業など）の登録・編集・削除
- 清掃者（フリーランス）の登録・編集・削除
- 各ユーザーへの LINE アカウント紐付け

#### 3.1.3 スケジュール管理
- 清掃案件の作成
  - 対象物件、清掃日時（日付・開始予定時刻）、担当清掃者のアサイン
- 作成したスケジュールの一覧表示（カレンダービュー / リストビュー）
- スケジュールの編集・削除

#### 3.1.4 通知送信

- スケジュール作成時に、担当清掃者と物件の通知有効な関係者へ LINE 通知を自動送信
- 清掃前日に清掃者へリマインド通知を自動送信
- 清掃完了時に物件の通知有効な関係者へ完了通知を自動送信

#### 3.1.5 清掃記録閲覧
- 全案件の清掃記録（完了時刻・所要時間）を一覧・詳細表示

#### 3.1.6 請求・支払い管理
- 清掃案件ごとの金額（民泊オーナーへの請求額・清掃者への支払い額）を記録
- 月次・期間別の集計表示
- CSV エクスポート機能

---

### 3.2 清掃者（LINE Mini App）

#### 3.2.1 スケジュール確認
- 自分がアサインされた清掃案件の一覧表示（日付順）
- 案件詳細の表示（物件名、住所、清掃日時、特記事項）

#### 3.2.2 清掃記録の登録
- 案件に対して以下を登録
  - 清掃開始時刻
  - 清掃完了時刻（自動で所要時間を算出）
  - メモ（任意）
- 登録後に業者へ完了通知が自動送信される

---

### 3.3 民泊オーナー（LINE Mini App）

#### 3.3.1 スケジュール確認
- 自物件に紐づいた清掃案件の一覧表示

#### 3.3.2 清掃記録の閲覧
- 清掃完了後の記録（完了時刻・所要時間・メモ）を閲覧

## 4. 通知仕様

| タイミング | 送信先 | 内容 |
|---|---|---|
| スケジュール作成時 | 清掃者 | 「〇月〇日 〇〇物件の清掃が入りました」+ 詳細リンク |
| スケジュール作成時 | 物件の通知有効な関係者 | 「〇月〇日に清掃が予定されました」+ 詳細リンク |
| 清掃前日 | 清掃者 | 翌日の清掃リマインド |
| 清掃完了時 | 物件の通知有効な関係者 | 「清掃が完了しました（完了時刻・所要時間）」 |
| 清掃完了時 | 業者 | 「〇〇さんが〇〇物件の清掃を完了しました」 |

## 5. 非機能要件

### 5.1 スケール

| フェーズ | テナント数 | 想定規模 | サーバー構成 |
| --- | --- | --- | --- |
| **開発・テスト** | 1社（自社） | 清掃者 5 名以下、物件 10 件以下 | Vercel Hobby + Supabase Free（無料） |
| **商用開始** | 2社〜 | 各社フリープラン内 | Vercel Pro + Supabase Pro（〜¥6,850/月） |
| **拡大** | 増加に応じて | 通知数・DB 使用量で判断 | 適宜プランアップ |

### 5.2 セキュリティ
- 管理 Web アプリはメール＋パスワード認証（業者のみ）
- ログインは業者別URL `/{slug}/login` のみ（汎用ログインは廃止）。ログイン時に所属テナントと URL の業者が一致するか検証する（他業者入口での誤ログイン防止）
- 未認証で管理画面にアクセスした場合・ログアウト時は、直近に使った業者の `/{slug}/login` へ戻す（slug が不明なら `/` の業者ID入力へ）
- LINE Mini App は LINE ログイン（LIFF）で本人確認
- 各ユーザーは自分に紐づくデータのみ閲覧可能（Supabase RLS によるテナント分離）

### 5.3 可用性
- 特別な SLA は定めない（通常の Web サービスレベル）
