# 本番反映手順書

> 対象：システム運営（ベンダー）担当者
> 構成：フロント = Vercel（Next.js）、バックエンド = Supabase（本番プロジェクト）
> 原則：**git の main ブランチへ push すると Vercel 本番が自動デプロイ**される。
> Supabase は push では反映されないため、**`supabase db push` 等を手動実行**する。

## 0. 前提・準備

- [ ] Supabase 本番プロジェクトを作成済み（プロジェクト参照 ID を控える）
- [ ] Supabase CLI でログイン済み：`supabase login`
- [ ] ローカルリポジトリで本番プロジェクトへリンク：
      `supabase link --project-ref <project-ref>`
- [ ] Vercel プロジェクト作成済み、Root Directory = `next`、Framework Preset = `Next.js`
- [ ] 本番の各種キーを Supabase ダッシュボード（Project Settings → API）で確認できる

実行順の推奨：**Supabase（DB → 認証フック → 拡張 → 関数 → cron）→ Vercel（環境変数 → デプロイ）→ 各業者のLINE設定 → 動作確認**。

---

## 1. Supabase（バックエンド）

### 1-1. マイグレーション適用

```bash
supabase db push
```

適用される主なマイグレーション（13本）：

- `20260620000000_init` … テーブル定義
- `20260620000001_rls` … RLS
- `20260620000002_auth_hook` / `20260621000000_fix_auth_hook` … JWTカスタムクレーム関数
- `20260621000001_grants_and_profile_policies`
- `20260621000002_company_slug`
- `20260621000003_service_role_grants`
- `20260621000004_daily_reminder_cron` … cron 雛形（コメント中心。実登録は後述）
- `20260622000000_viewer_role` … 閲覧者ロール
- `20260622000001_billing_flag` … 請求オプションフラグ
- `20260622000002_reminder_settings` / `20260622000003_reminder_matrix` … 定期送信設定＋重複防止ログ
- `20260622000004_protect_line_secrets` … LINEトークン/シークレットの列権限制限

> 確認：`supabase db push` 後、ダッシュボード → Database → Tables に各テーブルが存在すること。

### 1-2. 認証フック（JWTクレーム）を有効化 ★必須

ログイン時に `company_id` / `role` を JWT に載せる関数を有効化する。これを忘れると
**ログイン後に権限が効かない／会社ログインで弾かれる**。

- ダッシュボード → **Authentication → Hooks**
- **「Customize Access Token (JWT) Claims」** に `public.custom_access_token_hook` を割り当てて有効化

### 1-3. 拡張機能を有効化（定期送信を使う場合）

- ダッシュボード → **Database → Extensions** で `pg_cron`、`pg_net` を有効化

### 1-4. Edge Function をデプロイ（定期送信を使う場合）

```bash
supabase functions deploy daily-reminder
```

> `SUPABASE_URL` と `SUPABASE_SERVICE_ROLE_KEY` は Supabase が Edge Function に
> 自動で渡すため、関数用シークレットの手動設定は不要。

### 1-5. Cron ジョブを2本登録（定期送信を使う場合）

詳細は [ベンダー向け_定期リマインド有効化手順.md](./ベンダー向け_定期リマインド有効化手順.md) を参照。

| Name | Schedule(UTC) | 内容 | Body |
|---|---|---|---|
| `reminder-prev-day` | `0 11 * * *`（20:00 JST） | 前日 | `{"kind":"prev_day"}` |
| `reminder-same-day` | `0 23 * * *`（翌8:00 JST） | 当日 | `{"kind":"same_day"}` |

ダッシュボード → Integrations → Cron → Create job（Type = Edge Function `daily-reminder`）。

## 2. Vercel（フロントエンド）

### 2-1. 環境変数を設定

Vercel → Project → Settings → Environment Variables（Production）に以下を登録：

| 変数 | 値の取得元 |
|---|---|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase: Project Settings → API → Project URL |
| `SUPABASE_URL` | 同上（サーバー処理用。同じ値） |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabase: Project Settings → API → anon key |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase: Project Settings → API → service_role key（秘匿） |
| `NEXT_PUBLIC_LIFF_ID` | LINE Developers の本番 LIFF ID（例 `2010463195-prlgJV64`） |

> `SUPABASE_SERVICE_ROLE_KEY` は管理操作に使う秘密鍵。Production のみに設定し、漏らさない。

### 2-2. デプロイ（git push）

```bash
# main 以外で作業している場合はマージしてから
git push origin main
```

→ Vercel が自動でビルド＆本番デプロイ。完了後、本番URLで表示確認。

> 注意：push した瞬間に本番反映される。環境変数（2-1）と Supabase 側（1章）を
> **先に**済ませてから push すること。

### 2-3. LINE / LIFF の本番向け設定

- LIFF アプリのエンドポイント URL を本番ドメイン（`https://<本番>/liff`）に設定
- LINE Login のコールバック等、本番ドメインを許可

---

## 3. 各清掃請負業者ごとの設定（運用開始時）

1. 業者の管理者アカウントを発行（会社・管理者ユーザーを作成）
2. 管理者が **設定（LINE連携）** で自社のチャネルアクセストークン／シークレットを登録
3. 必要に応じて **設定（定期送信）** で送信先・タイミングを設定
4. 清掃者・オーナーを登録し、招待URLでLINE連携

---

## 4. 動作確認チェックリスト

- [ ] 本番URLにアクセスでき、`/<slug>/login` でログインできる
- [ ] ログイン後、権限（管理者／閲覧者）に応じた表示・操作制限が効く
- [ ] 物件・清掃者・オーナー・案件のCRUDが動く
- [ ] スケジュール作成／清掃完了で LINE 通知が届く（トークン登録済みの業者）
- [ ] LIFF（清掃者／オーナー）が本番で開ける
- [ ] 定期送信：`curl -X POST {FUNCTIONS_URL}/daily-reminder -H "Authorization: Bearer <service_role>" -d '{"kind":"prev_day"}'` で疎通確認
- [ ] 認証ユーザーが LINE トークン列を直接読めないこと（列権限）

---

## 5. ロールバック

- **フロント**：Vercel → Deployments → 直前の成功デプロイを「Promote to Production」
- **DB**：`supabase db push` は基本前進のみ。問題時は打ち消しマイグレーションを追加して再 push
  （`drop column` 等は事前にバックアップ／影響確認を）
- **Cron**：`select cron.unschedule('reminder-prev-day');` 等で停止

---

## 補足：環境ごとの違い

| | ローカル | 本番 |
|---|---|---|
| マイグレーション | `supabase db reset` | `supabase db push` |
| 認証フック | 自動（config） | ダッシュボードで手動有効化（1-2） |
| Cron 自動実行 | 不可（手動 curl） | Cron UI 登録で実行 |
| 環境変数 | `next/.env.local` | Vercel 環境変数 |
