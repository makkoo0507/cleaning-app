# ベンダー向け：業者アカウント発行手順

> 対象：システム運営（ベンダー）担当者
> 目的：新しい業者と、その管理者アカウントを発行する。

## 概要

運営専用画面 **`/vendor`** から、業者と管理者を一括で作成できる。
作成後、業者の管理者は **`/<slug>/login`** からログインして自社の管理画面を使える。

- 運営ログイン：`/vendor/login`（運営アカウントのみ。`is_platform_admin = true`）
- 運営アカウントはどの業者にも属さない（`contractor_id = null`）。

## 発行手順

1. `https://<本番ドメイン>/vendor/login` を開き、運営アカウントでログイン。
2. 「新規発行」フォームに入力：

   | 項目 | 説明 |
   |---|---|
   | 会社名 | 業者の表示名（例：ビーチ清掃） |
   | ログインURL（slug） | 英小文字・数字・ハイフンのみ。`/<slug>/login` になる（例：`beach`） |
   | プラン | free / paid |
   | 管理者名 | 業者管理者の表示名 |
   | 管理者メールアドレス | ログインID。重複不可 |
   | 管理者パスワード | 8文字以上。初回パスワード（業者へ安全に共有） |

3. 「発行する」を押す。成功すると上部に「業者を作成しました。ログインURL: /<slug>/login」と表示される。
4. 業者へ **ログインURL（`/<slug>/login`）・メール・初回パスワード**を伝える。
   （業者側は初回ログイン後、自社でメール変更・パスワード運用を行う）

## 作成されるもの

- 認証ユーザー（メール／パスワード、メール確認済み）
- `contractors`（業者・slug・プラン）
- `users`（role = `contractor_admin`、その業者に紐付け）

> 途中でエラーになった場合は自動でロールバックされる（作りかけのデータは残さない）。

## エラーと対処

| 表示 | 原因 |
|---|---|
| 未入力の項目があります | 必須項目が空 |
| ログインURL（slug）は英小文字… | slug に使えない文字 |
| パスワードは8文字以上に… | パスワードが短い |
| このログインURLは既に使われています | slug 重複 |
| このメールアドレスは既に登録されています | メール重複 |

## 運営アカウント自体の作り方（初回ブートストラップ・本番）

`/vendor` を使うには、最初の運営アカウントが1つ必要。本番では以下で作成する。

1. Supabase ダッシュボード → **Authentication → Users → Add user** で、
   運営用のメール／パスワードのユーザーを作成（Auto Confirm User を有効に）。
2. 作成したユーザーの **User UID** をコピー。
3. SQL Editor で、`public.users` に運営プロフィールを追加：

   ```sql
   insert into public.users (id, contractor_id, role, name, is_platform_admin)
   values ('<コピーしたUID>', null, 'contractor_admin', '運営担当', true);
   ```

4. `/vendor/login` から、そのメール／パスワードでログインできることを確認。

> ローカル開発では seed に運営アカウントを用意済み：`vendor@example.com` / `password123`。
