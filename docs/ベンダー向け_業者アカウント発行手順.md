# ベンダー向け：業者アカウント発行手順

> 対象：システム運営（ベンダー）担当者  
> 目的：新しい業者と、その管理者アカウントを発行する。

## 概要

運営専用画面 **`/vendor`** から、業者・管理者・ベンダーアカウントを一括で作成できる。  
作成後、業者の管理者は **`/<slug>/login`** からログインして自社の管理画面を使える。

- 運営ログイン：`/vendor/login`（`role = platform_admin` のアカウントのみ）
- 運営アカウントはどの業者にも属さない（`contractor_id = null`）

## 発行手順

1. `https://<本番ドメイン>/vendor/login` を開き、運営アカウントでログイン。
2. 「新規発行」フォームに入力：

   **業者情報**

   | 項目 | 説明 |
   |---|---|
   | 業者名 | 業者の表示名（例：ビーチ清掃） |
   | ログインURL（slug） | 英小文字・数字・ハイフンのみ。`/<slug>/login` になる（例：`beach`）。`vendor` は予約済みで使用不可 |
   | プラン | free / paid |

   **業者の管理者アカウント**

   | 項目 | 説明 |
   |---|---|
   | 管理者名 | 業者管理者の表示名 |
   | メールアドレス | ログインID。重複不可 |
   | パスワード | 8文字以上。業者へ安全に共有 |

3. 「発行する」を押す。成功すると「業者を作成しました。ログインURL: /<slug>/login」と表示される。
4. 業者へ **ログインURL・メール・初回パスワード** を伝える。

## 作成されるもの

| 種別 | 内容 |
|---|---|
| 業者レコード | `contractors`（業者名・slug・プラン） |
| 管理者アカウント | `users`（role = `contractor_admin`） |
| ベンダーアカウント | `users`（role = `contractor_vendor`）メール = `{運営メール}+{slug}@{domain}`、パスワード = `VENDOR_PASSWORD_BASE+{slug}` |

ベンダーアカウントは自動生成される。パスワードは `/vendor` のパスワード再設定から変更可能。

## エラーと対処

| 表示 | 原因 |
|---|---|
| 未入力の項目があります | 必須項目が空 |
| ログインURL（slug）は英小文字… | slug に使えない文字 |
| このログインURLは予約済みのため… | `vendor` などの予約語 |
| パスワードは8文字以上に… | パスワードが短い |
| このログインURLは既に使われています | slug 重複 |
| このメールアドレスは既に登録されています | メール重複 |

## 運営アカウント自体の作り方（初回ブートストラップ・本番）

1. Supabase ダッシュボード → **Authentication → Users → Add user** で運営用ユーザーを作成（Auto Confirm User を有効に）。
2. 作成したユーザーの **User UID** をコピー。
3. SQL Editor で `public.users` に運営プロフィールを追加：

   ```sql
   insert into public.users (id, contractor_id, role, name)
   values ('<コピーしたUID>', null, 'platform_admin', '運営担当');
   ```

4. `/vendor/login` からログインできることを確認。

> ローカル開発では seed に運営アカウントを用意済み：`vendor@example.com` / `password123`
