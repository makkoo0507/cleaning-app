## BF-00｜LINE 公式アカウント開設・Token 登録

アプリで LINE 通知を使い始める前に1回だけ実施する。

```
業者管理者
  │
  ├─ 1. LINE 公式アカウントを開設
  │       └─ LINE Business Manager（business.line.me）でアカウントを作成
  │
  ├─ 2. Messaging API チャネルを作成
  │       └─ LINE Developers Console（developers.line.biz）でプロバイダーを作成
  │               └─ 「新規チャネル作成」→「Messaging API」を選択して作成
  │
  ├─ 3. 認証情報を取得
  │       ├─ Channel Secret:「チャネル基本設定」タブに表示
  │       └─ Channel Access Token:「Messaging API 設定」タブ → 「発行」ボタンで取得
  │
  └─ 4. 管理 Web アプリから Token を登録
          └─ 設定画面で Channel Access Token・Channel Secret を入力して保存
                  └─ contractors テーブルに保存される
```

> LINE 通知（BF-03, BF-04）は Token が登録されていない業者には送信されない。

## BF-01｜初期セットアップ

業者が初めてアプリを使い始める際のセットアップ手順。LINE 紐付けの詳細は BF-02 を参照。

```
業者管理者
  │
  ├─ 1. 物件を登録
  │       └─ 物件名・住所・特記事項を入力
  │
  ├─ 2. 物件関係者を登録
  │       ├─ ユーザー情報（名前）を登録 → invite_token 発行
  │       ├─ 物件への紐付け・役割（owner / operations / sales）・通知設定
  │       └─ 招待 URL を LINE や SMS で関係者本人に送付
  │
  ├─ 3. 清掃者を登録
  │       ├─ ユーザー情報（名前）を登録 → invite_token 発行
  │       └─ 招待 URL を LINE や SMS で清掃者本人に送付
  │
  └─ 4. 社員を登録（必要な場合）
          └─ メール・パスワードでアカウント発行
```

## BF-02｜LINE アカウント紐付け

物件関係者・清掃者が初回利用時に LINE アカウントを紐づける。

> **前提（LINE アカウントの二層構造）**: ログインに使う LIFF は**ベンダー共通**、通知の送信元は**各業者の公式アカウント**。
> そのため「LIFF 紐付け（本人確認）」と「業者公式アカウントの友だち追加（通知受信）」は**別の作業**で、両方そろって初めて通知が届く。詳細は [要件定義書.md](要件定義書.md) 2.3 を参照。

```
業者管理者
  └─ 1. 管理 Web で招待 URL を発行・コピー
          └─ URL 例: https://example.vercel.app/liff/invite?token={invite_token}

  └─ 2. 招待 URL を LINE または SMS で相手に送付（アプリ外）
          └─ あわせて「自社の LINE 公式アカウントの友だち追加 URL/QR」も案内する

対象者（清掃者 または 物件関係者）
  └─ 3. 招待 URL をタップ → ロールに応じた LINE Mini App（LIFF）が起動

  └─ 4. LINE でログイン（LIFF 認証＝ベンダー共通）

  └─ 5. システムがトークンを照合
          ├─ 一致: users.line_user_id を保存 → invite_token を NULL クリア
          └─ 不一致 / 期限切れ: エラー表示

  └─ 6. 担当業者の LINE 公式アカウントを友だち追加（通知受信に必須）
          └─ LIFF の友だち追加機能では業者ごとのアカウントは追加できないため別導線

  └─ 7. 紐付け＋友だち追加 完了 → 以降は業者公式アカウントから LINE 通知が届く
          └─ 日常は通知メッセージ内のリンク（liff.line.me/{LIFF ID}）から Mini App を開く
```

## BF-03｜スケジュール登録

チェックアウト日が確定したタイミングで清掃案件を登録する。

```
業者（管理者 または 社員）
  │
  ├─ 1. スケジュール作成
  │       ├─ 対象物件・清掃日・開始予定時刻を入力
  │       └─ 清掃者をアサイン（後でアサインも可）
  │
  └─ 2. 作成完了 → 自動通知
          │
          ├─ 清掃者へ LINE 通知
          │       「〇月〇日 〇〇物件の清掃が入りました」+ 詳細リンク
          │
          └─ 物件の通知有効な関係者へ LINE 通知
                  「〇月〇日に清掃が予定されました」+ 詳細リンク
```

## BF-04｜清掃実施

清掃当日の記録登録から完了通知までの流れ。

```
【前日】
  システム（自動）
    └─ 清掃者へ LINE リマインド通知
            「明日 〇〇物件の清掃があります」

【当日】
  清掃者（LINE Mini App）
    │
    ├─ 1. マイスケジュールで案件・物件情報を確認
    │
    ├─ 2. 清掃開始 → 開始時刻を登録
    │
    ├─ 3. 清掃作業
    │
    └─ 4. 清掃完了 → 完了時刻・メモを登録
            │
            └─ 完了 → 自動通知
                    │
                    ├─ 物件の通知有効な関係者へ LINE 通知
                    │       「清掃が完了しました（完了時刻・所要時間）」
                    │
                    └─ 業者へ LINE 通知
                            「〇〇さんが〇〇物件の清掃を完了しました」

【完了後】
  物件関係者（LINE Mini App）
    └─ 清掃記録（完了時刻・所要時間・メモ）を閲覧

  業者（管理 Web）
    └─ 清掃記録を一覧で確認
```

## BF-05｜請求・支払い管理

月次または任意のタイミングで請求・支払いを管理する。

```
業者管理者（管理 Web）
  │
  ├─ 1. 案件ごとに請求額・支払い額を記録
  │       ├─ 民泊オーナーへの請求額
  │       └─ 清掃者への支払い額
  │
  ├─ 2. 月次集計を確認
  │       └─ 期間を指定して合計を表示
  │
  └─ 3. CSV エクスポート
          └─ 経理処理・支払い手続きを手動で実施
```
